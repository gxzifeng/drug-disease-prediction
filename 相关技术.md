# 相关技术

## 2.1 技术栈概述

本课题的系统架构采用典型的前后端分离与分层设计思想：后端以 Python 3.10+ 为基础，选用 FastAPI 作为高性能 Web 框架，结合 SQLAlchemy、MySQL 与 Redis 组成数据访问与缓存层，并通过 Celery 支持长耗时训练任务的异步调度；前端基于 Vue 3 与 TypeScript 实现组件化界面，配合 Element Plus 与 ECharts 构建表单管理、表格展示与多维可视化模块，Axios 与 Pinia 分别承担 HTTP 通信与状态管理职责。在算法层面，系统使用 Node2Vec 等网络表示学习方法获取药物与疾病节点的稠密向量表示，再通过随机森林与 XGBoost/LightGBM 等集成学习模型完成关联预测，形成“异质网络建模—表示学习—集成分类器—Web 系统展示”的完整技术链路。

## 2.2 FastAPI 后端框架

Web 后端开发中常用框架包括 Django、Flask 与 FastAPI 等。Django 提供较为完备的一体化解决方案，适合高度模板化的业务系统，但在精细控制异步 I/O 与高并发接口时灵活性有限；Flask 结构轻量，扩展自由度较高，但在类型约束、自动文档和统一依赖管理方面依赖额外库支持。FastAPI 基于 ASGI 与协程模型构建，原生支持异步路由处理与高并发请求调度，并利用类型注解自动生成 OpenAPI 文档与数据校验逻辑，在接口演化与多人协作场景中更便于维护。该框架在依赖注入、认证授权、中间件扩展及与 SQLAlchemy、Redis 的集成方面提供统一抽象，能够有效支撑药物‑疾病关联预测系统中涉及训练任务管理、图数据查询与预测服务的多模块后端需求。

## 2.3 SQLAlchemy ORM 框架

关系型数据持久化可以通过原生 SQL、轻量 ORM（如 Peewee）或功能完备的 ORM（如 SQLAlchemy、Django ORM）实现。原生 SQL 虽然执行路径透明，但在多表关联、分页统计及模式演化时容易形成难以维护的大量样板代码；Django ORM 与其框架强绑定，不利于在独立 FastAPI 工程中灵活复用。SQLAlchemy 同时提供声明式 ORM 与 Core 查询接口，可在对象化建模与精细 SQL 优化之间灵活切换；其会话管理、连接池与事务控制机制能够在高并发访问下保障数据一致性与性能稳定。结合 MySQL 与异步会话支持，SQLAlchemy 适合表达药物、疾病、图结构、嵌入结果和实验记录等多实体、多关系的数据模型，为后续扩展到更大规模数据和更复杂分析任务奠定坚实的数据层基础。

## 2.4 Vue 3 前端框架

前端开发领域常用框架包括 Angular、React 与 Vue 3。Angular 在模块系统、依赖注入和表单管理方面提供高度集成能力，适合大型企业级应用，但学习曲线陡峭且配置成本较高；React 通过虚拟 DOM 与 Hooks 提供灵活的函数式组件模式，在构建交互复杂的单页面应用方面具有优势，但在路由、状态管理与表单校验等方面需要引入多种周边库。Vue 3 在保持较小体积与易用性的同时，引入组合式 API 与更完备的 TypeScript 支持，在可读性、重构友好性和类型安全之间实现平衡。其生态中的 Vue Router、Pinia 与 Element Plus 能够无缝集成，形成适用于后台管理系统的完整解决方案，有利于实现图管理、训练任务管理与预测结果展示等多个功能视图的解耦与复用。

## 2.5 Node2Vec 网络表示学习算法

网络表示学习方法包括基于随机游走的 DeepWalk、Node2Vec，以及基于图神经网络的 GCN、GraphSAGE 等。DeepWalk 采用无偏随机游走获得节点序列，在局部结构刻画方面具有一定优势，但在全局结构与社区关系建模上的调节能力有限；GCN 与 GraphSAGE 在端到端任务上表现突出，却依赖较多标注数据和显著计算资源，工程部署成本较高。Node2Vec 在随机游走策略中引入 p、q 两个超参数，可在类 BFS 与类 DFS 行为之间平滑调节，使模型既能关注局部邻域结构，又能捕捉跨社区的潜在联系，适合药物‑疾病异质网络中多类型边与多尺度结构共存的场景。该算法在无监督设置下即可生成稳定的低维向量，易于与随机森林、XGBoost 等传统机器学习模型结合，兼顾预测性能与训练效率。

## 2.6 集成学习分类器（随机森林与梯度提升）

在药物‑疾病关联预测这一非线性、多特征任务中，常用分类模型包括逻辑回归、支持向量机、深度神经网络以及随机森林、梯度提升树等集成学习方法。逻辑回归在特征与标签关系近似线性时具有良好可解释性，但难以刻画复杂交互；支持向量机在中小规模数据上表现稳健，却在大样本与高维稠密特征场景中的训练成本较高。深度神经网络具备强大的表征能力，但对样本规模、超参数调优和推理资源提出更高要求。随机森林通过多棵决策树的 Bagging 集成提升鲁棒性和泛化能力，同时可以输出特征重要性，有利于分析药物与疾病嵌入在预测中的贡献；XGBoost/LightGBM 等梯度提升模型在处理稀疏特征、类别不平衡和大规模数据集时具有优良的收敛速度和性能表现，适合作为本系统的核心基准与对比模型。

## 2.7 小结

综上，本课题在后端选用 FastAPI 联合 SQLAlchemy、MySQL 与 Redis 构建高并发、可扩展的数据服务层，在前端采用 Vue 3 及其生态实现组件化、可视化的人机交互界面，在算法层将 Node2Vec 网络表示学习与随机森林、梯度提升模型相结合，形成方法与工程协同的技术体系，为药物‑疾病关联预测系统的稳定运行与后续算法扩展提供了坚实支撑。
