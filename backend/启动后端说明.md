# 启动后端说明

## 一、Docker 报错含义

你看到的错误：

```
unable to get image 'mysql:8.0': error during connect: Get "http://%2F%2F.%2Fpipe%2FdockerDesktopLinuxEngine/...": 
open //./pipe/dockerDesktopLinuxEngine: The system cannot find the file specified.
```

**含义：**  
- 系统在通过管道连接 **Docker 引擎** 时失败。  
- `//./pipe/dockerDesktopLinuxEngine` 是 Windows 上 Docker Desktop 和引擎通信用的管道。  
- “The system cannot find the file specified” 表示 **Docker Desktop 没有在运行**（或未安装），所以找不到这个管道。

**处理：**  
- 若已安装 Docker Desktop：先 **启动 Docker Desktop**，等右下角/托盘图标显示为“运行中”，再执行 `docker-compose up`。  
- 若未安装：需先安装 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)，再按上面步骤操作。

---

## 二、方式一：用 Docker 启动（推荐）

1. **启动 Docker Desktop**，等待其完全就绪。
2. 在项目根目录执行：
   ```powershell
   cd d:\cursor
   docker-compose up -d --build
   ```
3. 后端会随容器一起启动，API 默认在 **http://localhost:8000**。

---

## 三、方式二：本机直接跑 Python（不用 Docker）

需要本机已安装并运行 **MySQL 8** 和 **Redis**，且已建好数据库。

**注意：** 若你本机是 **Python 3.14**，很多依赖（如 FastAPI）可能尚无兼容的安装包，会报 `No matching distribution found for fastapi==...`。此时建议：
- **优先用方式一（Docker）** 启动后端，或  
- 使用 **Python 3.10 或 3.11** 创建虚拟环境后再安装依赖。

### 1. 安装 Python 依赖

在 **PowerShell** 里（不要用 `&&`，用分号 `;`）：

```powershell
cd d:\cursor\backend
pip install -r requirements.txt
```

### 2. 配置数据库

- 在 MySQL 中创建数据库，例如：
  ```sql
  CREATE DATABASE drug_disease_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
  ```
- 在 `d:\cursor\backend` 下新建或编辑 `.env`，设置：
  ```ini
  MYSQL_HOST=localhost
  MYSQL_PORT=3306
  MYSQL_USER=root
  MYSQL_PASSWORD=你的密码
  MYSQL_DB=drug_disease_db
  REDIS_HOST=localhost
  REDIS_PORT=6379
  ```

### 3. 初始化表结构（首次）

```powershell
cd d:\cursor\backend
alembic upgrade head
```

### 4. 启动后端

```powershell
cd d:\cursor\backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

看到类似 “Uvicorn running on http://0.0.0.0:8000” 即表示后端已启动。

---

## 四、小结

- **Docker 报错**：先启动 Docker Desktop，再执行 `docker-compose up`。  
- **启动后端**：任选其一 —— 用 Docker（方式一）或本机 MySQL+Redis+uvicorn（方式二）。
