# 系统使用说明

本系统用于**药物-疾病关联预测**：从数据上传、构图、嵌入训练到模型训练与预测的完整流程。以下说明如何按顺序使用各功能，以及常见问题。

---

## 一、使用流程概览

```
数据集管理 → 图管理（构建图） → 训练管理（训练嵌入） → 模型管理 → 实验 / 预测
```

1. **数据集管理**：上传药物-疾病关联数据（CSV 等），系统解析为药物、疾病及关联表。
2. **图管理**：选择数据集，构建异质图（划分训练/验证/测试边），得到「已构建」的图。
3. **训练管理**：选择已构建的图，选择算法（如 Node2Vec），新建训练任务；任务在后台执行，可在列表中查看状态与进度。
4. **模型管理**：基于已完成的嵌入训练任务，训练分类模型（如随机森林、XGBoost），用于预测药物-疾病关联。
5. **预测 / 实验**：使用已训练模型做单对预测、批量预测、药物推荐、疾病推荐，并可查看预测历史。

---

## 二、各功能入口与操作要点

| 步骤 | 菜单位置 | 主要操作 |
|------|----------|----------|
| 1. 数据 | **数据集管理** | 上传 CSV（需包含药物、疾病及关联信息），等待解析完成。 |
| 2. 构图 | **图管理** | 选择数据集与划分比例 → 点击「构建」→ 等待状态变为「已构建」。 |
| 3. 训练嵌入 | **训练管理** | 点击「新建训练任务」→ 选择**已构建**的图、算法（Node2Vec/GCN）、维度等 → 提交后接口立即返回，训练在后台进行。 |
| 4. 训练模型 | **模型管理** | 选择已完成的嵌入任务，训练分类器（随机森林/XGBoost 等）。 |
| 5. 预测 | **预测** | 单对预测、批量预测、推荐（疾病→药物 / 药物→疾病）、历史记录。 |

**注意：**

- 新建训练任务时，**图**必须为「已构建」；若图列表为空或没有已构建项，请先到「图管理」完成构图。
- 训练任务提交后会**立即返回**，训练在后台执行；请在「训练管理」列表刷新查看状态，由「训练中」变为「已完成」后再进行后续模型训练。
- 训练可能需数分钟，请勿关闭页面直到确认任务已创建；删除「训练中」的任务会移除记录，后台可能仍在算，但结果不再保留。

---

## 三、关于「删除」训练任务

- **之前删不掉的原因**：处于「训练中」的任务被设计为不可删除，以防误删。但若任务因超时/异常一直卡在「训练中」，就会无法删除。
- **当前逻辑**：已支持**删除「训练中」的任务**。删除时会出现提示：删除后该记录会移除，后台训练可能仍在进行但结果将不再保留。
- **建议**：仅对确认无用或长期卡在「训练中」的任务使用删除；正常等待完成的任务无需删除。

## 四、启动与访问

- **后端**：`docker-compose up` 或参见 `backend/启动后端说明.md`。
- **前端**：在 `frontend` 目录执行 `npm run dev`，浏览器访问控制台给出的地址（如 `http://localhost:5173`）。
- 首次使用如需管理员账号，可参考 `backend/启动后端说明.md` 或 `create_admin.py` 创建。
