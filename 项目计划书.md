# 基于网络表示学习的药物-疾病关联预测系统设计与实现 - 项目计划书

## 摘要

本课题围绕药物‑疾病关联预测系统的设计与实现，构建集数据管理、异质网络建模、网络表示学习与结果可视化于一体的综合平台。首先基于公开数据库整合药物、疾病及其已知关联，构建同时包含药物‑药物相似性、疾病‑疾病相似性及药物‑疾病关系的异质图，并完成数据清洗与标准化。其次采用Node2Vec等网络表示学习方法获取药物与疾病的低维向量表示，结合节点度中心性等网络结构特征形成多源融合特征表示。进一步引入随机森林、XGBoost等集成学习模型，对潜在药物‑疾病关联进行预测，并通过交叉验证、ROC曲线、AUC及AUPR等指标系统评估模型的泛化性能和鲁棒性。在系统实现层面，后端基于FastAPI与SQLAlchemy提供高并发REST服务和持久化存储，前端采用Vue 3 与 ECharts 实现可交互的网络可视化、预测结果展示及推荐解释。预期研究成果可为药物重定位与临床辅助决策提供实用工具，并为网络表示学习方法在生物医药领域的工程化落地提供参考。

## 一、项目概述

### 1.1 项目背景
药物-疾病关联预测是精准医疗和药物重定位（Drug Repurposing）领域的核心问题。传统的实验方法成本高、周期长，利用人工智能和网络表示学习技术可以从大规模生物医学数据中挖掘潜在的药物-疾病关联关系，为药物研发提供重要参考。

### 1.2 研究目标
- 构建药物-疾病异质网络，整合多种生物医学数据源
- 基于网络表示学习算法提取药物和疾病的低维向量表示
- 利用机器学习分类器预测潜在的药物-疾病关联关系
- 开发可视化的Web系统，支持交互式查询和结果展示

### 1.3 技术栈
**后端技术：**
- Python 3.10+
- FastAPI（RESTful API框架）
- SQLAlchemy（ORM数据库操作）
- MySQL（关系型数据库）

**前端技术：**

- NumPy、Pandas（数据处理）

### 1.4 研究价值与意义
本课题在方法层面将异质网络建模、网络表示学习与集成学习算法有机结合，以Node2Vec嵌入和随机森林等模型为核心，实现对药物‑疾病潜在关联的精细刻画与高效预测，丰富了基于网络表示学习的药物信息学研究范式。在工程实现层面，课题将上述算法封装为可配置的后端服务，并通过可视化前端实现从数据上传、图构建、嵌入训练到预测分析的一体化流程，为人工智能算法在生物医药领域的落地提供了可复用的系统框架。系统同时支持预测结果的可视化解释与交互式探索，有助于提升药物重定位与临床辅助决策的可理解性和可操作性，具有一定的理论价值与应用推广意义。

### 1.5 相关关键技术

#### 1.5.1 FastAPI 后端框架
Web 后端开发常用框架包括 Django、Flask 与 FastAPI 等。Django 提供“一站式”解决方案，内置 ORM、模板和管理后台，但框架较为厚重，难以精细控制高并发与异步 I/O 场景；Flask 结构轻量，扩展灵活，但在类型约束、自动文档和异步支持方面依赖第三方组件。FastAPI 基于 Starlette 与 Pydantic，原生支持 ASGI 与协程编程，在高并发请求、长耗时训练任务与实时接口中具备更优延迟表现；同时通过 Python 类型注解自动生成 OpenAPI 文档，降低接口维护成本。在安全机制、依赖注入、中间件扩展和与 SQLAlchemy、Redis 的集成方面也提供统一抽象，更适合支撑药物‑疾病关联预测系统中多模块、强类型、易扩展的后端架构。

#### 1.5.2 SQLAlchemy ORM 框架
关系型数据库访问可采用原生 SQL、轻量 ORM（如 Peewee）或功能完备的 ORM（如 SQLAlchemy、Django ORM）。原生 SQL 在复杂查询与多表关联时易产生大量样板代码，维护成本高；Django ORM 与框架强绑定，不利于在独立的 FastAPI 工程中复用。SQLAlchemy 提供成熟的 ORM 与 Core 两层抽象，在模型映射、关系建模、懒加载与会话管理方面具有较高灵活性，可在声明式建模与手写 SQL 之间平衡性能与可维护性。其对异步会话、连接池、事务控制和多数据库适配提供完备支持，能够满足药物、疾病、图结构与实验结果等多张表的复杂查询及分页统计需求，为后续扩展到更大规模数据集提供可靠的数据访问基础。

#### 1.5.3 Vue 3 前端框架
前端开发主流框架包括 Angular、React 与 Vue 3。Angular 功能完备且适合大型企业级项目，但依赖注入、装饰器和模块系统增加了学习与配置复杂度；React 通过虚拟 DOM 和 Hooks 提供灵活的组件化能力，但在路由、状态管理和表单处理等方面需引入多种周边库，工程搭建成本相对较高。Vue 3 采用组合式 API 与响应式数据模型，在类型支持、代码组织和可维护性方面实现改进，同时保持较小体积和较低使用门槛。其生态中的 Vue Router、Pinia 与 Element Plus 能够无缝协作，适合构建包含图可视化、训练管理、预测结果展示等多页面的后台系统。综合性能表现、生态完整度与学习成本，Vue 3 更契合药物‑疾病关联预测平台对快速迭代和复杂交互界面的需求。

#### 1.5.4 Node2Vec 网络表示学习算法
网络表示学习方法包括 DeepWalk、Node2Vec 以及基于图神经网络的 GCN、GraphSAGE 等模型。DeepWalk 采用无偏随机游走，易于实现但在局部与全局结构平衡方面调节能力有限；GCN 类方法在端到端任务上表现突出，但对标注数据和算力资源依赖较强，工程落地成本较高。Node2Vec 通过引入 p、q 两个超参数控制随机游走的 BFS/DFS 偏好，可在结构等价性与社区相似性之间灵活折中，适合在药物‑疾病异质网络中同时捕捉局部相似和跨模块关系。该算法在无监督设置下即可生成稳定的低维向量，便于与传统机器学习模型（如随机森林、XGBoost）衔接，实现特征可解释、计算代价可控的关联预测流程。

#### 1.5.5 集成学习分类器（随机森林与梯度提升）
药物‑疾病关联预测任务可采用逻辑回归、支持向量机、深度神经网络及集成学习模型等多种分类器。逻辑回归在线性可分数据上表现稳定，但难以刻画复杂非线性关系；深度神经网络参数规模大，对样本量和调参经验要求较高。随机森林通过多棵决策树的 bagging 集成，在处理高维特征和非线性边界时具有较强鲁棒性与抗过拟合能力，可同时给出特征重要性评估，便于解释药物或疾病嵌入在预测中的作用。XGBoost/LightGBM 等梯度提升模型在处理稀疏特征、类别不平衡和大规模数据方面表现优异。将 Node2Vec 等嵌入表示与随机森林、XGBoost 结合，能够在保证训练效率的前提下取得较高 AUC/AUPR 指标，适合作为药物‑疾病关联预测系统的基线与对比模型。

## 三、适合项目的算法方案

### 3.1 网络表示学习算法（核心算法）

#### 3.1.1 Node2Vec
- **原理**：基于Word2Vec的Skip-gram模型，通过有偏随机游走生成节点序列，学习节点的低维向量表示
- **优势**：可调节游走策略（BFS/DFS平衡），捕获网络的局部和全局结构
- **适用场景**：药物-疾病异质网络的节点嵌入
- **实现难度**：中等，有现成的库（gensim/node2vec）

#### 3.1.2 DeepWalk
- **原理**：通过随机游走生成节点序列，使用Skip-gram模型学习向量表示
- **优势**：简单高效，适合大规模网络
- **适用场景**：作为对比基线算法
- **实现难度**：低，易于实现

#### 3.1.3 Graph2Vec（可选，提升难度）
- **原理**：学习整个子图的向量表示
- **优势**：可以考虑子图结构信息
- **实现难度**：较高

### 3.2 机器学习分类算法

#### 3.2.1 随机森林（Random Forest）⭐推荐
- **原理**：集成多个决策树，通过投票或平均进行预测
- **优势**：
  - 能够处理非线性关系
  - 特征重要性分析
  - 对过拟合有较好的鲁棒性
  - 可解释性较好
- **适用场景**：作为主要的分类器，将网络嵌入向量和网络特征作为输入
- **实现难度**：低，scikit-learn直接支持
- **超参数**：n_estimators（树的数量）、max_depth（最大深度）、min_samples_split（最小分割样本数）

#### 3.2.2 XGBoost（Extreme Gradient Boosting）⭐推荐
- **原理**：梯度提升决策树，通过迭代优化损失函数
- **优势**：
  - 性能优秀，在许多竞赛中表现突出
  - 支持正则化，防止过拟合
  - 特征重要性分析
  - 处理缺失值
- **适用场景**：作为对比分类器，与随机森林进行性能比较
- **实现难度**：低，XGBoost库直接支持
- **超参数**：learning_rate、max_depth、n_estimators、subsample

#### 3.2.3 逻辑回归（Logistic Regression）
- **原理**：线性分类模型，使用sigmoid函数将输出映射到[0,1]
- **优势**：简单快速，可解释性强，作为基线模型
- **实现难度**：极低
- **适用场景**：作为简单基线，对比其他复杂模型的效果

#### 3.2.4 支持向量机（SVM）
- **原理**：寻找最优分离超平面
- **优势**：在高维空间表现良好
- **实现难度**：低
- **适用场景**：可选，作为对比实验

### 3.3 特征工程方法

#### 3.3.1 网络结构特征
- **度中心性（Degree Centrality）**：节点的连接数
- **接近中心性（Closeness Centrality）**：节点到其他所有节点的平均距离
- **介数中心性（Betweenness Centrality）**：节点在最短路径上出现的频率
- **聚类系数（Clustering Coefficient）**：节点的邻居节点之间的连接密度
- **PageRank值**：基于随机游走的节点重要性评分

#### 3.3.2 相似度特征
- **Jaccard相似度**：基于共同邻居计算药物-疾病对相似度
- **余弦相似度**：基于嵌入向量计算相似度
- **路径特征**：药物和疾病之间的最短路径长度

#### 3.3.3 组合特征
- **拼接嵌入向量**：将药物嵌入向量和疾病嵌入向量拼接
- **哈达玛积（Hadamard Product）**：向量逐元素相乘
- **平均/求和**：向量平均或求和

### 3.4 评估指标

#### 3.4.1 分类指标
- **准确率（Accuracy）**：正确预测的样本比例
- **精确率（Precision）**：预测为正例中实际为正例的比例
- **召回率（Recall）**：实际正例中被正确预测的比例
- **F1-Score**：精确率和召回率的调和平均
- **AUC-ROC**：ROC曲线下面积，衡量分类器整体性能
- **AUC-PR**：PR曲线下面积，适合不平衡数据集

#### 3.4.2 网络嵌入评估
- **链路预测**：使用嵌入向量预测边，评估嵌入质量
- **节点分类**：使用嵌入向量进行分类任务

### 3.5 数据处理算法

#### 3.5.1 数据预处理
- **缺失值处理**：均值填充、中位数填充、删除
- **异常值检测**：IQR方法、Z-score方法
- **数据标准化**：Min-Max标准化、Z-score标准化
- **类别编码**：One-Hot编码、标签编码

#### 3.5.2 采样策略（处理不平衡数据）
- **SMOTE（Synthetic Minority Oversampling Technique）**：合成少数类样本
- **随机欠采样**：随机删除多数类样本
- **代价敏感学习**：调整不同类别的权重

## 六、近三年相关研究现状（简要综述）

近三年，国内学者在基于网络表示学习的药物相关关系预测方面取得了系列进展。Wang 等提出 MGAVAEMDA 模型，将改进的图注意变分自编码器与随机森林结合用于微生物‑药物关联预测，证明深度表征与集成学习融合可显著提升 AUC 和 AUPR，并通过 PubMed 文献验证多数预测结果 [1]。Kuang 等在 BMC Bioinformatics 发表基于图注意网络与双层随机森林的微生物‑药物关联预测模型，强调在异质网络中联合考虑多源相似性与局部拓扑信息，从而提升对潜在关联的捕获能力 [2]。东北林业大学等提出 GCGACNN 模型，使用图神经网络提取嵌入后由随机森林完成分类，在多个基准数据集上整体优于传统机器学习方法 [3]。总体来看，现有工作多聚焦于微生物‑药物或药物‑靶标层面，尚缺乏面向临床疾病表型、具备可视化与交互分析能力的一体化药物‑疾病关联预测系统，这为本课题从系统设计与实现层面开展研究提供了空间。
综上所述，国内外研究表明，将图神经网络、注意力机制与随机森林等集成学习方法相结合，能够在复杂生物医学网络上取得优于传统方法的预测效果，但多数工作聚焦于方法本身的精度提升，对数据流程管理、系统化实现与可视化分析关注不足。围绕药物‑疾病关联的端到端系统仍相对匮乏，缺少将网络表示学习算法与工程化平台深度融合的实践案例。因此，构建集数据管理、网络表示学习、机器学习预测与交互式可视化于一体的药物‑疾病关联预测系统，不仅可在工程层面补齐现有研究的短板，也为后续算法创新与临床应用提供可验证的实验平台和真实应用场景。

## 五、技术难点与解决方案

### 5.1 数据不平衡问题
**问题**：已知的药物-疾病关联数量远少于未知关联，导致数据不平衡。
**解决方案**：
- 使用SMOTE算法合成少数类样本
- 采用代价敏感学习，调整类别权重
- 使用AUC-PR指标评估（比AUC-ROC更适合不平衡数据）

### 5.2 网络规模大导致的性能问题
**问题**：大规模网络（数万节点）的嵌入学习和可视化性能问题。
**解决方案**：
- 使用采样技术（随机采样节点子集）
- 使用批量处理优化
- 前端网络可视化使用虚拟化技术（只渲染可见区域）
- 使用Web Worker进行后台计算

### 5.3 模型训练时间长
**问题**：网络嵌入学习和模型训练需要较长时间。
**解决方案**：
- 使用Celery异步任务处理
- 实现训练进度实时反馈（WebSocket或轮询）
- 提供训练任务取消功能
- 使用模型缓存，避免重复训练

### 5.4 特征维度高
**问题**：网络嵌入向量维度较高，可能导致维度灾难。
**解决方案**：
- 使用PCA降维
- 使用特征选择算法（基于重要性）
- 使用正则化技术（L1/L2正则化）

## 八、风险评估与应对

### 8.1 技术风险
- **风险**：算法效果不理想
- **应对**：提前进行算法验证实验，准备备选方案（不同算法组合）

### 8.2 数据风险
- **风险**：公开数据质量不佳或格式不统一
- **应对**：准备多个数据源，实现灵活的数据预处理管道

### 8.3 时间风险
- **风险**：开发时间紧张，无法按时完成
- **应对**：优先实现核心功能，次要功能可适当简化或延后

### 8.4 性能风险
- **风险**：系统响应慢，用户体验差
- **应对**：提前进行性能测试，优化关键路径，使用缓存和异步处理
